# 竞品信息采集渠道清单

## 一手信息（直接体验）

| 渠道 | 获取内容 | 操作建议 |
|------|---------|---------|
| 注册试用 | 功能、体验、Onboarding | 录屏并截图关键页面 |
| 官网 | 定位、定价、功能列表 | 保存定价页截图 |
| 帮助中心 | 功能文档、API文档 | 看功能边界和限制 |
| 更新日志 | 迭代重心、发展方向 | 近3个月的 Changelog |
| 社区/论坛 | 用户反馈、痛点 | 搜高频问题 |

## 二手信息（公开数据）

| 渠道 | 获取内容 | 地址/工具 |
|------|---------|----------|
| SimilarWeb | 流量、来源、地区分布 | similarweb.com |
| App Annie / Sensor Tower | 下载量、排名趋势 | data.ai |
| G2 / Capterra | 用户评分、评价 | g2.com / capterra.com |
| Product Hunt | 产品发布、社区反馈 | producthunt.com |
| Crunchbase | 融资、团队、估值 | crunchbase.com |
| App Store / Google Play | 评分、评论、更新频率 | 应用商店 |
| 小红书 / 即刻 | 中文社区口碑 | 搜索品牌名 |
| Twitter/X | 创始人动态、用户反馈 | 搜索品牌名和创始人 |
| LinkedIn | 团队规模、招聘方向 | 搜公司主页 |

## 技术信息

| 渠道 | 获取内容 |
|------|---------|
| BuiltWith / Wappalyzer | 技术栈识别 |
| GitHub | 开源项目活跃度、Star 数 |
| 浏览器开发者工具 | API 结构、前端框架 |
| robots.txt / sitemap.xml | SEO 策略线索 |

## 信息采集优先级

当时间有限时，按此顺序采集：

1. **官网 + 定价页**（5分钟，获取定位和商业模式）
2. **注册试用核心流程**（15分钟，获取功能和体验）
3. **G2/应用商店评论**（10分钟，获取用户真实反馈）
4. **SimilarWeb 流量概览**（5分钟，获取增长渠道线索）
5. **更新日志/博客**（10分钟，获取迭代方向）

45 分钟内可以完成一个竞品的基础信息采集。
