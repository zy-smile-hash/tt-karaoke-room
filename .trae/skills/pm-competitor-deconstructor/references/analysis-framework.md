## 策略层深度拆解

### 定位分析框架

对每个竞品回答三个问题：
1. **给谁用？**（目标用户画像，越具体越好）
2. **解决什么问题？**（核心痛点，一句话）
3. **凭什么选你？**（差异化价值，与替代品的区别）

### 商业模式分析

| 模式 | 特征 | 代表产品 |
|------|------|---------|
| 纯SaaS订阅 | 月/年付费，按席位或用量 | Notion, Figma |
| Freemium | 免费版+付费增值 | Slack, Zoom |
| 交易抽成 | 平台抽取交易佣金 | Shopify, Stripe |
| 广告模式 | 免费使用，广告变现 | Google, 抖音 |
| 硬件+软件 | 硬件销售+软件订阅 | Apple, Peloton |
| 开源+商业版 | 社区免费+企业付费 | GitLab, Supabase |

### 护城河评估

对每项打分（0-5）：

- **网络效应**：用户越多产品越有价值？
- **数据壁垒**：积累的数据是否形成壁垒？
- **转换成本**：用户迁移走的成本有多高？
- **品牌认知**：品类心智占有率？
- **技术壁垒**：技术实现难度是否构成壁垒？
- **规模效应**：规模是否带来成本优势？

## 功能层深度拆解

### 功能树提取方法

1. 从导航结构提取一级模块
2. 进入每个模块提取二级功能
3. 对核心功能记录操作步骤
4. 标注付费墙位置

### 功能深度评级

| 等级 | 含义 | 判断标准 |
|------|------|---------|
| L1 基础 | 仅满足最低需求 | 能用但体验一般，无亮点 |
| L2 完整 | 覆盖主流场景 | 主流用例都能走通 |
| L3 优秀 | 超出预期 | 有创新交互或智能化处理 |
| L4 极致 | 品类标杆 | 被广泛引用为最佳实践 |

### 功能对标注意事项

- 不要只看"有没有"，要看"做到什么程度"
- 同名功能可能实现完全不同，要看具体表现
- 关注"隐性功能"：快捷键、批量操作、API、导入导出
- 记录功能更新频率，判断竞品投入重心

## 体验层深度拆解

### 体验走查流程

按以下路径逐一记录：

1. **落地页**（3秒测试：能否立即理解产品是什么）
2. **注册流程**（步数、必填信息量、社交登录支持）
3. **Onboarding**（引导方式、到Aha Moment的步数）
4. **核心任务**（完成主要任务的操作步数和流畅度）
5. **次要任务**（设置、账户管理、帮助中心）
6. **异常场景**（断网、空数据、权限不足、操作错误）

### 体验评分维度

| 维度 | 评分标准 |
|------|---------|
| 可用性 | 能否无障碍地完成核心任务 |
| 效率 | 完成任务需要多少步骤/时间 |
| 可学习性 | 新用户多快能上手 |
| 容错性 | 犯错后多容易恢复 |
| 愉悦度 | 使用过程中的情感体验 |

每项 1-5 星，3 星为行业平均水平。

### 截图标注建议

关键页面建议截图标注：
- 首页/落地页
- 注册/登录页
- 产品主界面（核心功能页）
- 定价页
- 设置页
- 空状态 / 错误状态

## 增长层深度拆解

### 获客渠道识别方法

| 渠道 | 如何识别 |
|------|---------|
| SEO | 搜索品类关键词看排名；用 SimilarWeb 看流量来源 |
| 内容营销 | 是否有博客/教程/模板库/社区 |
| 社媒 | Twitter/LinkedIn/小红书/即刻等活跃度 |
| PLG | 产品内是否有邀请/分享/协作等自传播机制 |
| 付费投放 | Google Ads Transparency / Meta Ad Library |
| KOL合作 | YouTube/B站/播客是否有评测内容 |
| 合作伙伴 | 集成/插件市场/API生态 |

### 付费转化分析

重点记录：
- 免费版能做什么？限制在哪里？
- 付费墙出现在哪个节点？（注册后/使用N次后/特定功能）
- 价格锚点设计（几个档位、推荐哪个、年付折扣）
- 试用期策略（有没有、多长、到期后什么体验）

### 留存机制识别

| 机制 | 表现 |
|------|------|
| 数据锁定 | 用户在产品内积累的数据越多越难走 |
| 习惯形成 | 日常工作流中的嵌入程度 |
| 社交绑定 | 团队/协作让个人无法独立离开 |
| 内容沉淀 | 用户创建的内容形成粘性 |
| 通知召回 | 邮件/推送的召回策略 |
